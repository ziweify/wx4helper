# 项目重命名指南：永利系统 -> YongLiSystem

## 📋 重命名步骤

由于 PowerShell 在处理中文路径时存在问题，请按照以下步骤手动执行重命名：

### 方法1：使用 Python 脚本（推荐）

1. **执行 Python 脚本**：
   ```bash
   python rename_project.py
   ```

2. **使用 Git 提交更改**：
   ```bash
   # 添加新文件
   git add YongLiSystem/
   
   # 删除旧文件
   git rm -r 永利系统/
   
   # 提交更改
   git commit -m "重命名项目: 永利系统 -> YongLiSystem"
   ```

### 方法2：手动重命名（如果 Python 不可用）

#### 步骤1：重命名文件夹

在文件资源管理器中：
- 将 `永利系统` 文件夹重命名为 `YongLiSystem`

#### 步骤2：重命名项目文件

在 `YongLiSystem` 文件夹中：
- 将 `永利系统.csproj` 重命名为 `YongLiSystem.csproj`
- 将 `永利系统.csproj.user` 重命名为 `YongLiSystem.csproj.user`（如果存在）

#### 步骤3：更新解决方案文件

在 `Vx3Plus.sln` 中：
- 查找 `"永利系统"` 替换为 `"YongLiSystem"`
- 查找 `永利系统\永利系统.csproj` 替换为 `YongLiSystem\YongLiSystem.csproj`

#### 步骤4：更新代码中的命名空间

使用 Visual Studio 的"查找和替换"功能（Ctrl+Shift+H）：

1. **查找范围**：整个解决方案
2. **查找内容**：`namespace 永利系统`
3. **替换为**：`namespace YongLiSystem`
4. **全部替换**

重复以下替换：
- `using 永利系统` -> `using YongLiSystem`
- `永利系统.` -> `YongLiSystem.`（注意：这个需要小心，确保只替换命名空间引用）

#### 步骤5：使用 Git 提交

```bash
# 添加新文件
git add YongLiSystem/

# 删除旧文件（使用 git rm 保持历史记录）
git rm -r 永利系统/

# 或者使用 git mv（推荐，保持完整历史）
# 注意：如果文件夹已重命名，需要先恢复，然后使用 git mv
git mv 永利系统 YongLiSystem
git mv YongLiSystem/永利系统.csproj YongLiSystem/YongLiSystem.csproj
git mv YongLiSystem/永利系统.csproj.user YongLiSystem/YongLiSystem.csproj.user

# 提交更改
git commit -m "重命名项目: 永利系统 -> YongLiSystem"
```

## ✅ 验证清单

重命名完成后，请验证：

- [ ] 文件夹已重命名为 `YongLiSystem`
- [ ] 项目文件已重命名为 `YongLiSystem.csproj`
- [ ] 解决方案文件中的引用已更新
- [ ] 所有代码文件中的命名空间已更新
- [ ] 项目可以正常编译
- [ ] Git 历史记录已正确保留

## 🔍 需要更新的文件类型

- `.cs` - C# 源代码文件
- `.csproj` - 项目文件
- `.sln` - 解决方案文件
- `.md` - 文档文件（可选）
- `.ps1` - PowerShell 脚本（如果包含项目路径）

## ⚠️ 注意事项

1. **备份**：在执行重命名前，建议先提交当前更改或创建备份
2. **编译测试**：重命名后务必测试编译，确保所有引用正确
3. **Git 历史**：使用 `git mv` 而不是直接重命名，可以保留文件历史记录
4. **路径引用**：检查是否有硬编码的项目路径需要更新
