## 📂 目录结构

### 新的目录组织方式

```
./永利系统/项目说明/
├── 目录.md                    # 目录索引文件（必须维护）
├── README-命名规则.md         # 本文件

│   ├── 002-简略标题.md

├── YYMMDD/                    # 另一天的文件夹
│   ├── 001-简略标题.md
│   └── ...
└── ...
```

### 目录结构说明

- **根目录**: `./永利系统/项目说明/`
  - 存放规则文件和目录索引
  - 不直接存放说明文件

- **日期子文件夹**: `YYMMDD/`
  - 格式：年份后两位 + 月份 + 日期（6位数字）
  - 例如：`251219/`（2025年12月19日）、`251220/`（2025年12月20日）
  - 每天创建一个文件夹
  - 该文件夹内存放当天的所有说明文件

- **目录索引文件**: `目录.md`
  - 必须维护，记录所有文件的索引
  - 使用 Markdown 格式，按日期分组
  - 方便搜索和查找

## 📋 文件命名格式

### 新格式（v2.0）

```
NNN-简略标题.md
```

### 格式说明：

- **NNN**: 当日文件索引（3位数字，从001开始）
  - 第1个文件: `001`
  - 第2个文件: `002`
  - 第3个文件: `003`
  - 依此类推...

- **简略标题**: 文件内容的简短描述（10个字以内，中文）
  - 必须简短明了
  - 一眼能看出做了什么
  - 使用中划线 `-` 连接

- **扩展名**: `.md` 或 `.txt`

### 完整路径示例

```
./永利系统/项目说明/251220/001-Ribbon布局详解.md
./永利系统/项目说明/251220/002-多标签页示例.md
./永利系统/项目说明/251221/001-新功能开发.md
```

### 格式要求

```markdown

- [002-多标签页示例](251220/002-多标签页示例.md)
- [003-目录结构重组](251220/003-目录结构重组.md)
```

### 目录索引维护规则

1. **必须维护**: 每次创建新文件时，必须同步更新 `目录.md`
2. **按日期分组**: 使用 `## YYYY-MM-DD` 作为日期标题
3. **使用链接**: 使用 Markdown 链接格式，方便跳转
4. **按索引排序**: 同一日期内的文件按索引号排序
5. **日期倒序**: 最新日期在最前面

### 目录索引的优势

- ✅ **快速查找**: 使用 Markdown 的标题和链接，方便导航
- ✅ **全文搜索**: Markdown 文件支持全文搜索
- ✅ **版本控制**: Git 可以追踪目录变化
- ✅ **自动索引**: 可以使用工具自动生成

## 📂 文件位置规则

### 创建新文件时的步骤

1. **确定日期**: 获取当前日期，格式化为 `YYMMDD`
2. **检查文件夹**: 检查 `./永利系统/项目说明/YYMMDD/` 是否存在
   - 不存在 → 创建文件夹
   - 存在 → 继续
3. **确定索引**: 检查该日期文件夹内已有文件数量
   - 已有 N 个文件 → 新文件索引为 `N+1`（补零到3位）
4. **创建文件**: 在日期文件夹内创建 `NNN-简略标题.md`
5. **更新目录**: 在 `目录.md` 中添加新文件链接

### 完整路径示例

```
创建日期: 2025-12-20
已有文件: 2个（001, 002）
新文件: ./永利系统/项目说明/251220/003-新功能说明.md
```

## 📌 重要提示

1. **目录结构**: 必须按日期创建子文件夹，不要直接在根目录创建文件
2. **目录索引**: 必须维护 `目录.md`，每次创建文件后更新
3. **标题必须简短**: 10字以内，不要啰嗦
4. **标题要有意义**: 一眼能看出内容
5. **使用中文标题**: 便于阅读识别
6. **保持一致性**: 同类型文件使用相似命名
7. **避免特殊字符**: 只使用中文、英文、数字、中划线
