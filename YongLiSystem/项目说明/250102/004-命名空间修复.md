# 命名空间修复说明

## 问题
编译错误：
```
error CS0234: 命名空间"Unit"中不存在类型或命名空间名"la"
```

## 原因
`Unit.la` 项目的实际命名空间是 `Unit.La`（大写的 La），而不是 `Unit.la`（小写的 la）。

## 修复

### BrowserTaskWindow.cs

**修复前**：
```csharp
using Unit.la.Controls;  // ❌ 错误：小写 la

private Unit.la.Controls.ScriptEditorControl? _scriptEditor;  // ❌
```

**修复后**：
```csharp
using Unit.La.Controls;  // ✅ 正确：大写 La

private ScriptEditorControl? _scriptEditor;  // ✅
```

## Unit.La 项目命名空间

根据实际代码，Unit.la 项目使用的命名空间是：
- `Unit.La.Controls` - 控件类
- `Unit.La.Scripting` - 脚本引擎类

注意：文件夹名是小写 `Unit.la`，但命名空间是 `Unit.La`（Pascal Case）。

## 编译结果
修复后，项目应该可以成功编译。✅
