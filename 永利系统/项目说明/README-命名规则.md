# 项目说明文件命名规则

## 📋 命名格式

```
YYMMDD-NNN-简略标题.md
```

### 格式说明：
- **YY**: 年份后两位（2位数字）
  - 2025年 = `25`
  - 2026年 = `26`

- **MMDD**: 月份和日期（4位数字）
  - 例如：12月19日 = `1219`
  - 例如：01月05日 = `0105`

- **NNN**: 当日文件索引（3位数字，从001开始）
  - 第1个文件: `001`
  - 第2个文件: `002`
  - 第3个文件: `003`
  - 依此类推...

- **简略标题**: 文件内容的简短描述（10个字以内，中文）
  - 必须简短明了
  - 一眼能看出做了什么
  - 使用中划线 `-` 连接

- **扩展名**: `.md` 或 `.txt`

## ⚙️ 规则要点

1. ✅ 年份格式为2位数字 `YY`（年份后两位）
2. ✅ 日期格式为4位数字 `MMDD`
3. ✅ 索引固定为3位数字 `NNN`，不足3位补0
4. ✅ 使用中划线 `-` 分隔各部分
5. ✅ 简略标题必须在10个字以内（中文）
6. ✅ 优先使用 `.md` 格式（Markdown）
7. ✅ 每天从 `001` 开始计数
8. ✅ 每次创建文件前检查当日已有文件数量
9. ✅ 自动递增索引号

### 不好的标题示例 ❌
```

❌ 修改  (太简略，看不出具体内容)
❌ 项目  (无意义)
❌ 临时文件  (不清晰)
❌ test  (使用英文且不清晰)
```

### 标题长度限制
```
✅ 5-8字: 最佳长度
⚠️ 9-10字: 可接受
❌ 超过10字: 必须精简
```
