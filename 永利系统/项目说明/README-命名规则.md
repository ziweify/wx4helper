# 项目说明文件命名规则

## 📋 命名格式

```
YYMMDD-NNN.md
```

### 格式说明：
- **YY**: 年份后两位（2位数字）
  - 2025年 = `25`
  - 2026年 = `26`

- **MMDD**: 月份和日期（4位数字）
  - 例如：12月19日 = `1219`
  - 例如：01月05日 = `0105`

- **NNN**: 当日文件索引（3位数字，从001开始）
  - 第1个文件: `001`
  - 第2个文件: `002`
  - 第3个文件: `003`
  - 依此类推...

- **扩展名**: `.md` 或 `.txt`

## 📝 示例

### 2025年12月19日的文件：
- 第一个文件: `251219-001.md`
- 第二个文件: `251219-002.md`
- 第三个文件: `251219-003.md`

### 2025年01月05日的文件：
- 第一个文件: `250105-001.md`
- 第二个文件: `250105-002.md`

### 2026年03月15日的文件：
- 第一个文件: `260315-001.md`
- 第二个文件: `260315-002.md`

## ⚙️ 规则要点

1. ✅ 年份格式为2位数字 `YY`（年份后两位）
2. ✅ 日期格式为4位数字 `MMDD`
3. ✅ 索引固定为3位数字 `NNN`，不足3位补0
4. ✅ 使用连字符 `-` 分隔日期和索引
5. ✅ 优先使用 `.md` 格式（Markdown）
6. ✅ 每天从 `001` 开始计数
7. ✅ 每次创建文件前检查当日已有文件数量
8. ✅ 自动递增索引号

## 📂 文件位置

所有项目说明文件统一存放在：
```
./永利系统/项目说明/
```

## 🔍 文件内容建议

每个说明文件应包含：
- 📅 创建日期
- 📌 说明主题
- 📝 详细内容
- 👤 创建者（可选）
- 🏷️ 标签（可选）
