## 📋 新命名规则

### 格式
```
YYMMDD-NNN-简略标题.md
```

### 详细说明
- **YY**: 年份后两位（如 2025 → `25`）
- **MMDD**: 月份和日期（如 12月19日 → `1219`）
- **NNN**: 当日索引（001, 002, 003...）
- **简略标题**: 10个字以内的简短描述

### 示例
```
251219-001-MVVM架构重构.md
251219-002-本地DLL引用.md
251219-003-引用缺失解决.md
```

## 📝 使用方法

### 创建新文件时
1. 检查当日已有文件数量
2. 确定下一个索引号（如当天有3个文件，新文件为004）
3. 创建简短标题（10字以内）
4. 按格式命名：`YYMMDD-NNN-标题.md`

### 示例
假设今天是 2025-12-20，已有2个文件：
```
251220-001-数据库集成.md
251220-002-用户权限管理.md
```

新建文件应命名为：
```
251220-003-报表功能实现.md
```

## ⚠️ 注意事项

1. **标题必须简短明了** - 让人一眼看出文件内容
2. **使用中文标题** - 便于快速识别
3. **避免特殊字符** - 只使用中文、英文、数字、中划线
4. **保持一致性** - 同类型文件使用相似的命名风格
