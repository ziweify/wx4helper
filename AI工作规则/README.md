## 📋 规则列表

### 1. 终端命令执行规则 ⭐⭐⭐
**文件**: `终端命令执行规则.md`  
**重要性**: 🔴 极高（长期反复出现的问题）  
**适用范围**: 所有涉及终端命令执行的操作

**核心要点**:
- ❌ 不要使用`run_terminal_cmd`处理中文路径
- ✅ 创建脚本文件让用户执行
- ✅ 提供多种执行方式
- ✅ 说明清楚为什么需要用户手动执行
- ⚠️ PowerShell脚本中的中文字符串使用Base64编码

**快速判断**:
```
命令/路径包含中文？
  是 → 创建脚本 + 让用户执行
  否 → 可以使用 run_terminal_cmd
```

### 2. 项目说明文件命名规则 ⭐⭐⭐
**文件**: `项目说明文件命名规则.md`  
**重要性**: 🔴 极高（必须严格遵守）  
**适用范围**: 所有在 `永利系统/项目说明/` 目录下创建的文件

**核心要点**:
- ✅ 按日期创建子文件夹（格式：`YYMMDD/`）
- ✅ 文件命名格式：`NNN-简略标题.md`（去掉日期前缀）
- ✅ 每次创建文件后必须更新 `目录.md`
- ✅ 标题控制在10字以内
- ❌ 不要在根目录直接创建文件

**快速判断**:
```
创建项目说明文件？
  是 → 检查日期文件夹 → 创建文件 → 更新目录.md
```

### 3. 代码实现原则 ⭐⭐⭐⭐⭐
**文件**: `代码实现原则.md`  
**重要性**: 🔴 极高（核心开发原则）  
**适用范围**: 所有代码实现和功能开发

**核心要点**:
- ✅ **优先使用现成实现**，不要重复造轮子
- ✅ 评估现有实现的适用性（功能、稳定性、复杂度）
- ✅ 只有在必要时才手动实现（无法满足需求或过于复杂）
- ❌ 不要盲目手动实现，先查找现有实现
- ❌ 不要因为"想学习"而重复造轮子

**快速判断**:
```
需要实现功能？
  是 → 查找现有实现（框架/库/项目内）
      │
      ├─ 有且适用 → ✅ 使用现成实现
      │
      ├─ 有但过于复杂 → ⚠️ 评估后决定
      │
      └─ 无或不适用 → ❌ 手动实现
```

**决策标准**:
- ✅ 功能完善且满足需求 → 使用现成
- ⚠️ 功能完善但过于复杂 → 评估后决定
- ❌ 功能不完善或无法满足需求 → 手动实现
- ❌ 需求非常简单 → 手动实现（轻量级）

### 4. UI界面设计器优先规则 ⭐⭐⭐⭐⭐
**文件**: `UI界面设计器优先规则.md`  
**重要性**: 🔴 极高（必须严格遵守）  
**适用范围**: 所有UI界面创建和修改

**核心要点**:
- ✅ **优先使用Visual Studio设计器创建和编辑UI**
- ✅ 在设计器中查看、设计、编辑、修改界面
- ✅ 尽可能在设计器中创建和展示所有控件
- ❌ 避免在代码中动态创建控件（除非必要）
- ❌ 不要忽略设计器，完全用代码构建UI

**快速判断**:
```
需要创建UI界面？
  是 → 控件是否支持设计器？
      │
      ├─ 是 → ✅ 使用设计器创建
      │
      ├─ 否 → ⚠️ 代码创建（但容器在设计器）
      │
      └─ 运行时动态 → ⚠️ 混合使用（容器在设计器，内容在代码）
```

**决策标准**:
- ✅ 静态UI控件 → 使用设计器
- ✅ 布局和容器 → 使用设计器
- ✅ 属性设置 → 使用设计器
- ⚠️ 运行时动态内容 → 代码创建（但容器在设计器）
