## 📋 常见问题

### Q1: 为什么需要手动输入验证码？

**A**: Yyds666 平台使用图片验证码防止自动化脚本。当前实现采用手动输入方式：
- **优点**: 简单、准确率100%
- **缺点**: 需要人工操作
- **适用**: 初期测试、低频登录

如需全自动登录，可以集成OCR服务（百度OCR、腾讯OCR等）。详见《新增平台对接指南》第5节。

---

### Q2: 登录超时怎么办？

**A**: 等待60秒未检测到URL跳转会显示"登录超时"。原因可能是：
1. 验证码未输入或输入错误
2. 登录按钮未点击
3. 登录失败（账号密码错误）

**解决方法**:
1. 确认验证码已输入
2. 确认登录按钮已点击
3. 检查用户名和密码是否正确
4. 查看浏览器是否有错误提示

---

### Q4: 余额显示不正确怎么办？

**A**: 余额查询功能框架已实现，但需要根据实际页面调整。

**调整步骤**:
1. 登录成功后，在浏览器中找到余额显示的元素
2. 打开 Chrome DevTools → Elements
3. 找到余额元素的 class 或 id（如 `<div class="balance">1000.00</div>`）
4. 修改 `Yyds666Script.cs` 的 `GetBalanceAsync()` 方法中的选择器

**示例**:
```csharp
// 查找余额元素（根据实际页面调整）
var balanceElement = document.querySelector('.balance') 
                  || document.querySelector('#userBalance')
                  || document.querySelector('[data-balance]');
```
