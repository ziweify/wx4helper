## 📋 需要您执行的操作

### 选项A：简单方法（推荐新手）

#### 步骤1：重命名文件夹
1. 打开文件资源管理器
2. 导航到 `E:\gitcode\wx4helper\`
3. 右键点击 `永利系统` 文件夹
4. 选择"重命名"
5. 输入新名称：`YongLiSystem`
6. 按回车确认

#### 步骤2：重命名项目文件（如果存在）
1. 进入 `YongLiSystem` 文件夹
2. 如果看到 `永利系统.csproj`，重命名为 `YongLiSystem.csproj`
3. 如果看到 `永利系统.csproj.user`，重命名为 `YongLiSystem.csproj.user`

#### 步骤3：提交到 Git
打开 PowerShell 或 Git Bash，执行：
```bash
cd E:\gitcode\wx4helper
git add .
git commit -m "重命名项目: 永利系统 -> YongLiSystem

- 重命名项目文件夹和文件
- 创建 Unit.la 脚本编辑器库
- 更新所有代码中的命名空间
- 解决中文路径问题"
```

### 选项B：Git 方法（推荐高级用户，保留完整历史）

打开 Git Bash 或 PowerShell，执行：
```bash
cd E:\gitcode\wx4helper

# 使用 git mv 重命名文件夹和文件
git mv "永利系统" "YongLiSystem"

# 进入新文件夹
cd YongLiSystem

# 重命名项目文件（如果存在）
if exist "永利系统.csproj" (
    git mv "永利系统.csproj" "YongLiSystem.csproj"
)
if exist "永利系统.csproj.user" (
    git mv "永利系统.csproj.user" "YongLiSystem.csproj.user"
)

# 返回根目录
cd ..

# 提交更改
git commit -m "重命名项目: 永利系统 -> YongLiSystem (保留历史)

- 使用 git mv 保持文件历史记录
- 创建 Unit.la 脚本编辑器库
- 更新所有代码中的命名空间
- 解决中文路径问题"
```

### 步骤4：验证（两种方法都需要）

1. **检查文件夹**：
   - 确认 `YongLiSystem` 文件夹存在

2. **检查项目文件**：
   - 确认 `YongLiSystem/YongLiSystem.csproj` 存在

3. **重新打开解决方案**：

   - 确认编译成功

## ❓ 常见问题

### Q1: Git mv 命令失败，提示找不到文件
**A**: 这是因为 Git Bash 在处理中文路径时有问题。请使用"选项A：简单方法"。

### Q2: 重命名后 Visual Studio 提示找不到项目
**A**: 关闭 Visual Studio，重新打开 `Vx3Plus.sln`。

### Q3: 编译时提示找不到 Unit.la
**A**: 
1. 在解决方案资源管理器中右键点击解决方案
2. 选择"还原 NuGet 包"
3. 重新生成解决方案

### Q4: 如何验证 Git 历史是否保留？
**A**: 使用以下命令：
```bash
git log --follow YongLiSystem/Program.cs
```
如果看到文件的完整历史（包括重命名前的提交），说明历史已保留。
