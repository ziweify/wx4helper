# 正确的视图创建语句
时间戳存储为 .NET DateTime.Ticks（整数格式）。使用以下 SQL 创建视图：
DROP VIEW IF EXISTS LogEntryFormatted;

CREATE VIEW LogEntryFormatted AS
SELECT 
    Id,
    datetime((Timestamp / 10000000.0) - 62135596800, 'unixepoch', 'localtime') AS 时间,
    Level,
    Source,
    Message,
    Exception,
    ThreadId,
    UserId,
    ExtraData
FROM LogEntry;



# 转换原理
Ticks 是从 0001-01-01 00:00:00 开始的 100 纳秒间隔数
Ticks / 10000000.0 = 转换为秒数
- 62135596800 = 减去从 0001-01-01 到 1970-01-01 的秒数差，得到 Unix 时间戳
'unixepoch' = 告诉 SQLite 这是 Unix 时间戳
'localtime' = 转换为本地时区（北京时间）
# 验证
# 执行以下查询验证转换是否正确：
SELECT 
    Id,
    datetime((Timestamp / 10000000.0) - 62135596800, 'unixepoch') AS 转换后的时间,
    datetime((Timestamp / 10000000.0) - 62135596800, 'unixepoch', 'localtime') AS 错误方式_多8小时,
    datetime('now') AS 当前系统时间,
    Level,
    Source,
    Message
FROM LogEntry
ORDER BY Id DESC
LIMIT 5;
如果 转换后的时间 显示为正确的日期时间格式（如 2024-01-01 12:00:00），说明转换成功。
